# Session Compression Workflow

Analyze the conversation and compress it following this systematic workflow:

## 1. Identify Learning Progress Milestones
- Topics covered during the session
- Concepts the student has mastered
- Skills practiced and demonstrated
- Progress made from previous sessions

## 2. Extract Student Questions and Concerns
- Unresolved issues that need follow-up
- Points of confusion or misunderstanding
- Questions asked but not fully answered
- Areas where clarification is needed

## 3. Preserve Action Items and Commitments
- Homework assignments given
- Practice exercises to complete
- Review tasks scheduled
- Goals set for next session
- Deadlines and timelines

## 4. Summarize Key Concepts Covered
- Main topics discussed
- Important definitions introduced
- Critical examples worked through
- Techniques and methods taught

## 5. Note Areas Needing Follow-Up
- Topics where student is struggling
- Incomplete explanations that need continuation
- Pending clarifications or deeper dives
- Prerequisites that need reinforcement

## 6. Maintain Emotional Context
- Student confidence level (low/medium/high)
- Engagement quality during session
- Frustration points encountered
- Motivation and enthusiasm indicators

## Preservation Priorities

**HIGH Priority** (always preserve):
- Learning progress and milestones
- Action items and commitments
- Unresolved questions

**MEDIUM Priority** (preserve when significant):
- Key concepts covered
- Student concerns and struggles
- Follow-up areas identified

**LOW Priority** (omit unless relevant):
- Small talk and greetings
- Off-topic discussions
- Repetitive confirmations

## Compression Target

Aim for a 10:1 compression ratio: 10 messages should compress into approximately 1 summary paragraph, while preserving all high-priority information and significant medium-priority elements.

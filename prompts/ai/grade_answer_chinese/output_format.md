# 中文答案评分输出格式

## JSON 结构规范

返回严格的 JSON 格式，包含以下字段：

```json
{
  "score": 0-100,
  "correct": true|false,
  "feedback": "string",
  "analysis": "string",
  "explanation": "string",
  "wrong_reason": "string"
}
```

## 字段说明

### score（分数）
- 类型：整数
- 范围：0-100
- 说明：根据答案的准确性、完整性、规范性综合评定
- 评分标准：
  - 90-100分：答案完全正确，表达规范，理解深刻
  - 80-89分：答案基本正确，有少量瑕疵或表达不够精准
  - 70-79分：答案部分正确，遗漏部分要点或存在理解偏差
  - 60-69分：答案勉强及格，存在明显错误或遗漏较多要点
  - 0-59分：答案错误或严重偏离题意

### correct（正误判断）
- 类型：布尔值
- 说明：整体判断答案是否正确
- 判定标准：
  - true：答案核心要点正确，无重大错误
  - false：答案存在重大错误、核心要点错误或严重偏离题意

### feedback（反馈意见）
- 类型：字符串
- 长度：100-300字
- 内容：针对学生答案的具体反馈，包括优点和不足
- 结构：
  1. **优点肯定**（如有）：简要肯定答案中的正确部分或亮点
  2. **问题指出**：明确指出答案中的错误或不足
  3. **改进建议**：提供具体的改进方向

示例：
```
你对文章主旨的把握基本准确，能够抓住关键信息。但在汉字书写上，"陪伴"的"陪"字右边应该是"咅"（pǒu），不是"部"；拼音标注中"重要"的"重"应读zhòng yào（去声+阴平），而非chóng yào。建议加强形声字的部件识记，注意多音字在不同语境中的读音变化。
```

### analysis（答案分析）
- 类型：字符串
- 长度：200-500字
- 内容：深入分析答案的各个方面，提供专业的评价
- 结构：
  1. **汉字与拼音**：分析汉字书写、笔顺、部首、拼音声调等
  2. **词汇与语义**：分析词语理解、成语运用、语义准确性等
  3. **语法与句式**：分析句子结构、语法规范、表达流畅性等
  4. **内容理解**：分析对题目要求的理解、要点覆盖情况等
  5. **文学鉴赏**（如适用）：分析对文学作品的理解和鉴赏能力

示例：
```
【汉字书写】"陪"字属于形声字，左边"阝"（阜）表意，右边"咅"表声。学生误写为"部"旁，说明对形声字的声旁识记不够准确。建议通过字源分析记忆：陪伴需要在旁边（阝），发音来自"咅"。

【拼音声调】"重要"的"重"是多音字，读chóng时表示"重复、再次"，读zhòng时表示"重量、重要"。此处应读zhòng（去声），学生标注为chóng，说明对多音字的语境辨析能力需要加强。

【内容理解】学生准确提取了文章的三个要点，概括能力较强。但在表述时使用了"非常重要"这样的口语化表达，建议在书面答题时使用"至关重要""意义重大"等更规范的书面语。

【整体评价】答案显示出良好的阅读理解能力，但在汉字书写和拼音标注的基础知识上存在薄弱环节，需要加强基本功训练。
```

### explanation（知识点讲解）
- 类型：字符串
- 长度：200-400字
- 内容：针对答案涉及的知识点进行详细讲解，帮助学生理解和掌握
- 结构：
  1. **核心知识点**：明确指出相关的语文知识点
  2. **详细讲解**：提供清晰、准确的知识讲解
  3. **示例说明**：通过例子帮助理解
  4. **记忆方法**：提供实用的记忆技巧或学习方法

示例：
```
【形声字的识记方法】
汉字中约80%是形声字，由表意的形旁和表音的声旁组成。"陪"字的形旁是"阝"（阜），本义与土山、高地有关，引申为"在旁边"；声旁是"咅"（pǒu），提示读音。

记忆技巧：
1. 形旁看意义："阝"在左表示与地形、位置有关，如陪、陈、陆、陵
2. 声旁看读音："咅"声的字多读pei/bei/pou音，如培、赔、陪、剖
3. 对比记忆：陪（阝+咅）≠ 部（立+阝+口），两者形似但结构不同

【多音字的辨析方法】
"重"字的两个读音：
- chóng（阳平）：重复、重新、重叠，如"重逢""重来""重峦叠嶂"
- zhòng（去声）：重量、重要、重视，如"重要""重点""举足轻重"

记忆口诀：重（chóng）复再来，重（zhòng）要认真。

建议：遇到多音字时，先判断词义，再确定读音。可以通过组词法巩固记忆，如"重（chóng）新重（zhòng）视"。
```

### wrong_reason（错误原因）
- 类型：字符串
- 长度：100-200字
- 内容：分析学生出错的根本原因，帮助找到学习的薄弱环节
- 结构：
  1. **直接原因**：指出具体的错误类型
  2. **深层原因**：分析导致错误的知识或能力缺陷
  3. **改进方向**：提出针对性的学习建议

示例：
```
【直接原因】汉字书写错误源于对形声字结构的混淆，将声旁"咅"误记为"部"；拼音错误源于多音字辨析不清，未能根据语境选择正确读音。

【深层原因】
1. 形声字知识掌握不牢固，未能准确识别形旁和声旁的功能
2. 多音字积累不足，缺乏根据语境判断读音的能力
3. 基础知识训练不够系统，对汉字结构和语音系统的理解较为薄弱

【改进方向】
1. 系统学习形声字的构字规律，建立形旁表意、声旁表音的认知
2. 整理常用多音字，通过组词和例句掌握不同读音的使用语境
3. 加强汉字书写和拼音标注的专项训练，夯实语文基础
```

## 注意事项

1. **JSON 格式严格**：必须返回有效的 JSON，所有字符串使用双引号，特殊字符正确转义
2. **字段完整性**：所有6个字段都必须包含，不能遗漏
3. **内容针对性**：反馈和分析必须针对具体答案，避免空泛的套话
4. **专业性**：体现中文语言教育的专业性，提供准确的知识讲解
5. **建设性**：不仅指出错误，更要提供改进方法和学习建议
6. **字数控制**：各字段内容应在规定字数范围内，既要充分又要精炼
7. **汉字规范**：涉及汉字分析时，准确说明笔画、部首、结构
8. **拼音标注**：提供拼音时必须包含完整的声母、韵母和声调标记
9. **鼓励为主**：在指出问题的同时，注意肯定学生的优点，保持积极的教育态度
